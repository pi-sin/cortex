# Skill: Prioritize

When deciding what to build next, use the ICE framework:

## Scoring

- **Impact** (1-10): How much does this move key metrics? (DAU, integrations connected, time saved, NPS)
- **Confidence** (1-10): Do we have user evidence? Is the tech proven?
- **Ease** (1-10): Days = high. Months = low.

**Score = (Impact × Confidence × Ease) / 10**

## Output

Ranked table:
| Rank | Item | Impact | Confidence | Ease | Score | Notes |

## Rules

- P0/P1 bugs skip the framework — always go first
- If two items score within 10%, pick retention over acquisition
- Building for 10 passionate users beats 100 lukewarm ones
- "Quick wins" (score >50, effort <1 day) → batch and ship weekly
- Never prioritize Phase 3+ features over Phase 1 gaps
