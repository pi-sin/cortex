# Skill: User Feedback Analysis

When processing beta feedback, reviews, or support tickets:

## 1. Categorize Each Piece

- **Bug** — something is broken
- **UX friction** — works but confusing/slow
- **Feature request** — they want something new
- **Integration issue** — auth, sync, or data problem with a service
- **Performance** — app slow, laggy, high memory
- **Praise** — what's working well (protect this)

## 2. Rate Severity (bugs and friction only)

- **P0** — blocking, can't use core functionality
- **P1** — painful, workaround exists but bad experience
- **P2** — annoying, doesn't block workflow
- **P3** — minor, cosmetic or edge case

## 3. Frequency Signal

3+ users reporting same issue = TREND. Flag it.

## 4. Action Items

For each actionable item:

- What to do (1 sentence)
- Where in codebase (file path)
- Effort estimate (hours)
- Priority recommendation

## 5. Response Draft

Short, human reply. Grateful, specific, no corporate speak. Under 4 sentences.
